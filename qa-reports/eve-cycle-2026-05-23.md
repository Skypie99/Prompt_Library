## What shipped on `cycle/auto-2026-05-23-eve`

### F-eve-1 — Sort prompts dropdown  ✅
- Native `<select>` in the All Prompts header: **Newest first** (default) · **A → Z** · **Most run**.
- Persisted to `promptlib:sort`. "Most run" reuses the runCounts map already gathered for the F-fast-2 usage badge — no new storage.
- **Alex fix:** trimmed redundant `sr-only` label span — the `<select>`'s `aria-label="Sort prompts"` was the actual accessible name; the wrapping `<label>` was double-naming.
- **Gary added:** `src/lib/__tests__/sort.test.ts` (~13 cases): load/save round-trip, default + SSR safety, sortPrompts immutability, each mode, tie-breaking.

### F-eve-2 — Tag chips show count  ✅
- Each tag chip displays "#analysis · 5". Counts come from a new `getTagsWithCounts()` in `prompts.ts`; the existing `getTags()` now delegates to it (one source of truth for ordering).
- **Alex fix #1:** inactive count text bumped from `text-ink-soft` to `text-ink-muted` — at 10px on cream, ink-soft was ~3.5:1 (under AA); ink-muted is ~6.7:1.
- **Alex fix #2:** added missing `focus-visible` rings on the "+N more" / "Show fewer" buttons (carryover oversight from F3 days).
- `aria-label` per chip spells out "Filter by #analysis, 5 prompts"; the count `<span>` is `aria-hidden` to avoid double-reading.
- **Gary added:** 5 cases for `getTagsWithCounts` in the existing `prompts.test.ts`.

### F-eve-3 — Resume pill in the hero  ✅
- "⏱ Resume → [last prompt title]" pill below the search button — only shown when `recentPrompts[0]` exists. One click opens it.
- **Alex fix:** the title `<span>` had `truncate` but no flex constraints → long titles overflowed the pill instead of clipping. Added `min-w-0 flex-1` so `truncate` actually clips, plus `shrink-0` on the icon/label/arrow children. Also added `w-full` to the button so `max-w-xl` behaves on narrow viewports.
- **Gary review:** pure presentational; no new pure-logic module; no test layer beyond the existing recent-list coverage. Noted.

### F-eve-4 — "Run again" in history rows  ✅
- Each history entry gets a coral "Run again" button next to "Restore". Click → restore the run's values into the form AND immediately re-run with them.
- Auto-hides while another run is in flight (parent passes `undefined` for `onRunAgain`).
- Required a small refactor in `PromptDetail`: extracted `runWithValues(values)` from the existing `handleRun` so the run logic accepts explicit values (avoids the stale-closure trap where `setValues` + `handleRun` would race against React's re-render). `handleRun` becomes a one-liner: `() => runWithValues(values)`.
- New `PlayIcon` in the icon set.
- **Alex fix (real UX one):** the trigger is in the History panel, which sits *below* the response panel. Clicking "Run again" would start a stream the user couldn't see without scrolling up. Added a `responsePanelRef` and a `requestAnimationFrame` → `scrollIntoView({behavior: 'smooth', block: 'start'})` on `handleRunAgain` so the new stream comes into view automatically.
- **Gary review:** the `runWithValues` extraction is behavior-preserving and paves the way for direct unit testing once RTL lands; no new lib module to test today. Noted.
