## What shipped on `fastloop/auto-2026-05-23`

5 small, visible, independent features. Each is one focused commit; reachable + accessible from the first commit; revert any single feature with one `git revert`.

### F-fast-1 — Char/token estimate by Run button (`4d16831`)

A tiny "~234 chars · ~58 tokens · Opus 4.7 · ⌘↵" readout in the PromptDetail footer that updates live as you type.

**See it:** open any prompt → type into a variable → watch the line under the Run button update.

**Likely refine:** none. **Likely remove:** none — pure visible info, zero new surface.

### F-fast-2 — "Run N×" usage badge on PromptCard (`00c08df`)

Quiet pill next to the category tag on each card, showing how many times you've run that prompt. Hidden when 0.

**See it:** open a prompt, run it once (with your API key), close the modal — the card shows "Run 1×".

**Likely refine:** if Sky wants to see "last run at X" instead of just count, the underlying `loadRuns` already has timestamps — small follow-up. **Likely remove:** unlikely; it's a quiet visual cue.

### F-fast-3 — Storage usage readout in Settings (`1e6393b`)

A small breakdown card under Backup & Restore showing where the user's `promptlib:*` localStorage budget is going, with a "Typical browser quota is ~5–10 MB per site" reassurance line.

**See it:** Settings → scroll to Backup & Restore → look below the import controls.

**Likely refine:** could surface a "trim run history" action if a user is near their cap. Not in v1. **Likely remove:** unlikely.

### F-fast-4 — "Copy template (with {{variables}})" (`0277223`)

The existing "Copy" button is now labeled "Copy filled" (primary); a new small "Copy template" text link below copies the raw body so you can paste the unfilled template elsewhere.

**See it:** open any prompt → look just below the Run button.

**Likely refine:** if more copy variants land (e.g. "Copy without category"), they'd cluster here. **Likely remove:** none expected.

### F-fast-5 — Compact / Comfortable grid density toggle (`356bdec`)

New header button (sits between the search button and the theme toggle) that flips the prompt grid between two layouts. Persisted to `promptlib:density`.

**See it:** click the grid icon in the header. Compact = sm:2 / md:3 / lg:4 with tighter padding and one-line descriptions. Comfortable = original layout.

**Likely refine:** could add a "List view" as a third mode later. **Likely remove:** unlikely — power users with 50+ prompts will reach for this.
