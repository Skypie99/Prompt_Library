## Forward-looking suggestions for quality posture

- **Treat `npm run typecheck && npm test` as the new ship gate** the moment
  the test runner lands. The morning's CI proposal already wires this in
  GitHub Actions, but adopting it as a personal habit before the CI is live
  is what actually changes day-to-day quality.
- **Co-locate one test file with every new pure-logic module from now on.**
  The pattern is now established — every existing pure module (10 files
  across the two projects) has at least one test. Keep that to one per new
  module and the safety net grows in step with the code.
